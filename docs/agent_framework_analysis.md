# 自律型AIエージェントフレームワーク（OpenClaw / CrewAI）の導入検討

Athena（現在のDiscord Bot）に対して、自律型AIエージェント機能を提供する **OpenClaw** または **CrewAI** などを導入する場合のメリット・デメリットをまとめました。

---

## 1. 概要: なぜエージェント機能が必要か？

現在の Athena は「1回の質問に対して1回回答する」「プロンプトから画像を生成する」という **受動的（Reactive）なチャットボット** です。
これら自律型エージェント（Autonomous Agent）フレームワークを統合すると、Athena は「目的を与えれば自ら計画を立て、検索し、コードを実行し、画像を生成して最終結果を報告する」という **能動的（Proactive）な自律アシスタント** に進化します。

---

## 2. 比較分析

### 案A: OpenClaw の導入

[OpenClaw](https://openclaw.ai/) は、ローカル環境での実行を前提とし、強力なシステムアクセス（シェル実行、ファイル操作、ブラウザ操作など100以上のスキル）を備えた、PCの「パーソナルアシスタント」としての性質が強いオープンソースAIエージェントです。

#### 🟢 メリット (Pros)
- **強力なスキルセット**: インストール直後から、Web検索、ファイルシステムの読み書き、ターミナルコマンドの実行といったPC操作が可能です。  
  *（例: 「サーバーのログを調べてDiscordにパッチの差分を送って」といった操作が完結します）*
- **ローカルファーストの哲学**: Athenaの「プライバシー完全保護・外部APIゼロ」の思想と完全に一致しています。Ollama経由のローカルLLMとも相性が良いです。
- **自律的稼働**: ユーザーが操作しなくても、バックグラウンドでの定期タスクや常駐監視などを自律的に行わせることができます。

#### 🔴 デメリット (Cons)
- **セキュリティ・リスクの激増**: Discord経由で誰でも（または第三者のプロンプトインジェクションによって）サーバーのシェルコマンドを実行できる状態になりかねず、致命的な脆弱性につながる可能性があります。
- **アーキテクチャの競合**: OpenClaw 自体がDiscord等のメッセージルーター機能を持っています。既存の `bot/cogs/` や `QueueService` といったAthenaのDiscord処理アーキテクチャを捨てるか、大きく設計を書き直す必要があります（画像生成パイプラインの統合が難航する可能性があります）。

---

### 案B: CrewAI の導入

[CrewAI](https://docs.crewai.com/) は、Pythonベースのマルチエージェント・オーケストレーション・フレームワークです。複数のAIエージェントに「役割（リサーチャー、ライター、エンジニアなど）」を与え、チームとして協力して複雑なタスクを解決させます。

#### 🟢 メリット (Pros)
- **Pythonライブラリとしての高い親和性**: 既存のAthenaコードにそのままインポートして使えます。アーキテクチャを壊さず、`/research` コマンドのような形で独立したCogとして追加できます。
- **柔軟なツール連携**: 現在の `ImageService` や `LLMService` を CrewAI の「カスタムツール」として登録するのが非常に簡単です。  
  *（例: シナリオライターエージェントが物語を考え、アートディレクターエージェントが最適なプロンプトを考案し、画像生成ツールを叩く、という連動がDiscord上で実現します）*
- **推論精度の向上**: Gemma 3 12Bのようなローカル特化LLMでも、「計画」「実行」「批評」の役割を分けることで、単一のプロンプト回答よりも遥かに高品質なアウトプット（コードや文章）を出せます。

#### 🔴 デメリット (Cons)
- **レスポンスタイムの劇的な悪化**: 複数のエージェントが何度もLLM（Ollama）を呼び出して議論するため、最終的なDiscordへの回答までに **数分〜10分以上** かかることがあります（DiscordのInteraction有効期限15分問題に直面しやすくなります）。
- **ローカルLLMの性能限界**: エージェントの自律思考（ReActプロンプトなど）には極めて高い推論能力が要求されます。GPT-4oやClaude 3.5では難なく動くCrewAIのタスクも、Gemma 3 12B（Ollama）だとループに陥ったり、指示を無視したりする確率が上がります。

---

## 3. 結論と推奨のロードマップ

**結論：Athenaの現行アーキテクチャに導入するなら「CrewAI」が圧倒的におすすめです。**

OpenClaw はそれ単体で主役となるアプリケーションであるため、Athena（Discordボット）の裏側に組み込むには向いていません（むしろボットシステムが乗っ取られます）。
一方で、CrewAI であれば Python パッケージとして組み込めるため、「必要なときだけ複数エージェントを召喚する」という使い方が可能です。

### 📌 おすすめの導入ステップ（CrewAIの場合）

1. **ツールのパッケージ化**
   - 現在の `image_service.py` の生成関数を `@tool` デコレータでラップし、CrewAIエージェントが使えるようにします。
2. **非同期アーキテクチャの拡充**
   - 推論に時間がかかるため、`/chat` のような一問一答ではなく、`/task` コマンドを作成し、進捗状況（「リサーチ中...」「画像生成中...」）を随時Discordにストリーミング（Embed更新）する仕組みを作ります。
3. **ローカルLLM専用のプロンプトチューニング**
   - Gemma 3 12B がエージェント・ループ（Thought -> Action -> Observation）を理解しやすいよう、CrewAI側のシステムプロンプトを日本語＆ローカル向けに調整します。
